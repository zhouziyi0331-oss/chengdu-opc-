# 成都OPC中心 - 完整部署指南

## 🎯 部署目标
将网站部署到免费云服务，国内可访问：
- 数据库：Supabase（免费500MB）
- 后端：Render（免费）
- 前端：Vercel（免费，国内可访问）

预计时间：20分钟

---

## 第一步：部署数据库（Supabase）⏱️ 5分钟

### 1.1 创建Supabase项目

1. 访问：https://supabase.com
2. 点击 "Start your project" → 使用GitHub账号登录
3. 点击 "New project"
4. 填写信息：
   - **Name**: `chengdu-opc`
   - **Database Password**: 设置一个强密码（例如：`OpcChengdu2024!`）**记住这个密码！**
   - **Region**: 选择 `Northeast Asia (Tokyo)` 或 `Southeast Asia (Singapore)`
   - **Pricing Plan**: Free（默认）
5. 点击 "Create new project"
6. 等待2-3分钟，项目创建完成

### 1.2 获取数据库连接URL

1. 在项目页面，点击左侧 "Project Settings"（齿轮图标）
2. 点击 "Database"
3. 找到 "Connection string" → 选择 "URI"
4. 复制连接字符串，格式类似：
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.xxx.supabase.co:5432/postgres
   ```
5. 将 `[YOUR-PASSWORD]` 替换为你刚才设置的密码
6. **保存这个URL**，后面会用到

### 1.3 执行数据库脚本

1. 在Supabase项目页面，点击左侧 "SQL Editor"
2. 点击 "New query"
3. 打开本地文件 `/Users/alwan/chengdu-ai-community/backend/schema.sql`
4. 复制全部内容，粘贴到SQL编辑器
5. 点击右下角 "Run" 按钮
6. 看到 "Success. No rows returned" 表示成功

---

## 第二步：部署后端（Render）⏱️ 5分钟

### 2.1 创建Render账号

1. 访问：https://render.com
2. 点击 "Get Started" → 使用GitHub账号登录
3. 授权Render访问你的GitHub仓库

### 2.2 创建Web Service

1. 在Render控制台，点击 "New +" → "Web Service"
2. 选择你的仓库：`zhouziyi0331-oss/chengdu-opc-`
3. 填写配置：
   - **Name**: `chengdu-opc-backend`
   - **Region**: `Singapore (Southeast Asia)` 或 `Oregon (US West)`
   - **Root Directory**: `backend`
   - **Environment**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `node src/app.js`
   - **Instance Type**: `Free`

### 2.3 配置环境变量

在 "Environment Variables" 部分，点击 "Add Environment Variable"，添加以下变量：

| Key | Value |
|-----|-------|
| `DATABASE_URL` | 粘贴第一步获取的Supabase连接URL |
| `JWT_SECRET` | `chengdu-opc-jwt-secret-2024` |
| `ADMIN_JWT_SECRET` | `chengdu-opc-admin-jwt-secret-2024` |
| `NODE_ENV` | `production` |
| `PORT` | `3000` |

### 2.4 部署

1. 点击 "Create Web Service"
2. 等待5-10分钟，Render会自动构建和部署
3. 部署成功后，会显示一个URL，类似：`https://chengdu-opc-backend.onrender.com`
4. **保存这个URL**，这是你的后端API地址

### 2.5 测试后端

访问：`https://chengdu-opc-backend.onrender.com/api/health`

如果看到 `{"status":"ok"}` 表示后端部署成功！

---

## 第三步：部署前端（Vercel）⏱️ 5分钟

### 3.1 创建Vercel账号

1. 访问：https://vercel.com
2. 点击 "Sign Up" → 使用GitHub账号登录
3. 授权Vercel访问你的GitHub仓库

### 3.2 导入项目

1. 在Vercel控制台，点击 "Add New..." → "Project"
2. 找到并选择 `chengdu-opc-` 仓库
3. 点击 "Import"

### 3.3 配置项目

1. **Framework Preset**: 选择 `Vite`
2. **Root Directory**: 点击 "Edit"，输入 `frontend`
3. **Build Command**: `npm run build`（默认）
4. **Output Directory**: `dist`（默认）

### 3.4 配置环境变量

在 "Environment Variables" 部分，添加：

| Name | Value |
|------|-------|
| `VITE_API_URL` | 粘贴第二步获取的Render后端URL（例如：`https://chengdu-opc-backend.onrender.com`） |

### 3.5 部署

1. 点击 "Deploy"
2. 等待2-3分钟，Vercel会自动构建和部署
3. 部署成功后，会显示一个URL，类似：`https://chengdu-opc-xxx.vercel.app`
4. **这就是你的网站地址！**

---

## 第四步：配置后端CORS ⏱️ 2分钟

### 4.1 更新Render环境变量

1. 回到Render控制台
2. 进入你的 `chengdu-opc-backend` 服务
3. 点击 "Environment"
4. 添加新的环境变量：

| Key | Value |
|-----|-------|
| `FRONTEND_URL` | 粘贴你的Vercel网站URL（例如：`https://chengdu-opc-xxx.vercel.app`） |

5. 点击 "Save Changes"
6. Render会自动重新部署后端（约2分钟）

## 📝 重要提示

### 1. Render免费版限制
- 15分钟无访问会自动休眠
- 首次访问需要等待30秒唤醒
- 建议：使用UptimeRobot等服务定时ping后端，保持活跃

### 2. 自定义域名（可选）
如果你有自己的域名，可以在Vercel设置：
1. Vercel项目 → Settings → Domains
2. 添加你的域名（例如：`opc.chengdu.com`）
3. 按照提示配置DNS记录

### 3. 数据备份
定期在Supabase导出数据库备份：
1. Supabase → Database → Backups
2. 点击 "Download" 下载备份

### 4. 监控和日志
- Render日志：Render控制台 → Logs
- Vercel日志：Vercel控制台 → Deployments → 点击部署 → Logs

## ❓ 常见问题

**Q: 网站打不开？**
A: 检查Vercel和Render的部署状态，确保都显示"Live"

**Q: 后端API报错？**
A: 检查Render的环境变量是否正确配置，特别是DATABASE_URL

**Q: 访问码登录失败？**
A: 确保在管理后台创建了访问码，并且访问码状态为"启用"

**Q: 图片无法上传？**
A: 当前版本使用URL方式，需要先将图片上传到图床（如：sm.ms、imgbb.com），然后粘贴URL

**Q: 想修改代码怎么办？**
A: 修改本地代码后，执行：
```bash
cd /Users/alwan/chengdu-ai-community
git add .
git commit -m "更新说明"
git push
```
Vercel和Render会自动重新部署
