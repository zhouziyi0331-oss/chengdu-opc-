### 3️⃣ 在服务器上部署后端（10分钟）

**连接到服务器**：
在阿里云控制台点击"远程连接" → "Workbench远程连接"

**运行自动部署脚本**：
```bash
# 下载并运行部署脚本
curl -o deploy.sh https://raw.githubusercontent.com/zhouziyi0331-oss/chengdu-opc-/main/backend/deploy-aliyun.sh
chmod +x deploy.sh
./deploy.sh
```

脚本会自动：
- ✅ 安装Node.js和PM2
- ✅ 克隆代码
- ✅ 安装依赖
- ✅ 配置环境变量
- ✅ 启动服务

**手动部署（如果脚本失败）**：
参考 `阿里云部署指南.md` 第三步

---

### 4️⃣ 部署前端到Vercel（5分钟）

1. 访问：https://vercel.com
2. 导入GitHub仓库：`chengdu-opc-`
3. 配置：
   - Framework: Vite
   - Root Directory: `frontend`
   - 环境变量：`VITE_API_URL` = `http://你的服务器IP:3000`
4. 点击Deploy

---

### 5️⃣ 配置域名（10分钟）

**在阿里云DNS控制台**（https://dns.console.aliyun.com）添加记录：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| CNAME | @ | cname.vercel-dns.com |
| CNAME | www | cname.vercel-dns.com |
| A | api | 你的服务器IP |

**在Vercel绑定域名**：
- Settings → Domains → 添加 `成都opc.top`

---

### 6️⃣ 配置HTTPS（可选但推荐，5分钟）

在服务器上运行：

```bash
# 安装Nginx和Certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# 复制Nginx配置
sudo cp ~/chengdu-opc-/backend/nginx-config.conf /etc/nginx/sites-available/chengdu-opc
sudo ln -s /etc/nginx/sites-available/chengdu-opc /etc/nginx/sites-enabled/

# 修改配置中的域名（如果需要）
sudo nano /etc/nginx/sites-available/chengdu-opc

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx

# 申请SSL证书
sudo certbot --nginx -d api.成都opc.top
```

完成后，更新Vercel环境变量：
- `VITE_API_URL` 改为 `https://api.成都opc.top`

## ❓ 遇到问题？

### 后端无法访问
```bash
# 检查服务状态
pm2 status
pm2 logs chengdu-opc-backend

# 检查防火墙
# 在阿里云控制台 → 防火墙 → 添加规则 → TCP 3000端口
```

### 前端无法连接后端
1. 检查Vercel环境变量 `VITE_API_URL` 是否正确
2. 检查后端 `.env` 中的 `FRONTEND_URL` 是否正确
3. 重新部署前端

### DNS未生效
- 等待10-30分钟
- 使用 `nslookup 成都opc.top` 检查
