## 第三步：部署后端到阿里云服务器 ⏱️ 15分钟

### 3.1 连接到服务器

在阿里云控制台，点击"远程连接" → "Workbench远程连接"

### 3.2 安装Node.js

```bash
# 更新系统
sudo apt update

# 安装Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node -v  # 应该显示 v18.x.x
npm -v   # 应该显示 9.x.x
```

### 3.3 安装PM2（进程管理器）

```bash
sudo npm install -g pm2
```

### 3.4 上传后端代码

**方法1：使用Git（推荐）**

```bash
# 安装Git
sudo apt install -y git

# 克隆仓库
cd ~
git clone https://github.com/zhouziyi0331-oss/chengdu-opc-.git
cd chengdu-opc-/backend

# 安装依赖
npm install
```

**方法2：手动上传**

如果Git克隆失败，可以：
1. 在本地打包backend文件夹为zip
2. 使用阿里云控制台的"文件上传"功能上传
3. 在服务器上解压：`unzip backend.zip`

### 3.5 配置环境变量

```bash
cd ~/chengdu-opc-/backend

# 创建.env文件
cat > .env << 'EOF'
DATABASE_URL=postgresql://postgres:zhouyue950331@db.xxx.supabase.co:5432/postgres
JWT_SECRET=chengdu-opc-jwt-secret-2024
ADMIN_JWT_SECRET=chengdu-opc-admin-jwt-secret-2024
NODE_ENV=production
PORT=3000
FRONTEND_URL=https://成都opc.top
EOF
```

**重要**：将上面的 `DATABASE_URL` 替换为你在第二步获取的Supabase连接字符串！

### 3.6 启动后端服务

```bash
# 使用PM2启动
pm2 start src/app.js --name chengdu-opc-backend

# 设置开机自启
pm2 startup
pm2 save

# 查看运行状态
pm2 status
pm2 logs chengdu-opc-backend
```

### 3.7 配置防火墙

在阿里云控制台：
1. 进入"轻量应用服务器控制台"
2. 点击你的服务器
3. 点击"防火墙"
4. 添加规则：
   - **应用类型**: 自定义
   - **协议**: TCP
   - **端口**: 3000
   - **策略**: 允许
5. 点击"确定"

### 3.8 测试后端

在浏览器访问：`http://你的服务器IP:3000/api/health`

如果看到 `{"status":"ok"}` 表示后端部署成功！

## 第五步：配置域名 ⏱️ 10分钟

### 5.1 在Vercel绑定域名

1. 在Vercel项目页面，点击 "Settings" → "Domains"
2. 输入你的域名：`成都opc.top`
3. 点击 "Add"
4. Vercel会显示需要配置的DNS记录

### 5.2 在阿里云配置DNS

1. 访问：https://dns.console.aliyun.com
2. 找到 `成都opc.top` 域名，点击"解析设置"
3. 添加以下记录：

**记录1：前端（Vercel）**
- **记录类型**: CNAME
- **主机记录**: @
- **记录值**: `cname.vercel-dns.com`
- **TTL**: 10分钟

**记录2：前端www（Vercel）**
- **记录类型**: CNAME
- **主机记录**: www
- **记录值**: `cname.vercel-dns.com`
- **TTL**: 10分钟

**记录3：后端API（阿里云服务器）**
- **记录类型**: A
- **主机记录**: api
- **记录值**: 你的服务器公网IP（例如：43.123.45.67）
- **TTL**: 10分钟

### 5.3 更新后端环境变量

回到阿里云服务器，更新FRONTEND_URL：

```bash
cd ~/chengdu-opc-/backend

# 编辑.env文件
nano .env
```

将 `FRONTEND_URL` 改为：
```
FRONTEND_URL=https://成都opc.top
```

保存后重启后端：
```bash
pm2 restart chengdu-opc-backend
```

### 5.4 更新前端环境变量

1. 回到Vercel项目页面
2. 点击 "Settings" → "Environment Variables"
3. 找到 `VITE_API_URL`，点击"Edit"
4. 改为：`https://api.成都opc.top`
5. 点击 "Save"
6. 点击 "Deployments" → 最新的部署 → "Redeploy"

### 5.5 配置Nginx反向代理（可选，推荐）

为了让API使用HTTPS和标准端口，在服务器上安装Nginx：

```bash
# 安装Nginx
sudo apt install -y nginx

# 安装Certbot（免费SSL证书）
sudo apt install -y certbot python3-certbot-nginx

# 创建Nginx配置
sudo nano /etc/nginx/sites-available/chengdu-opc
```

粘贴以下内容：

```nginx
server {
    listen 80;
    server_name api.成都opc.top;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/chengdu-opc /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx

# 申请SSL证书
sudo certbot --nginx -d api.成都opc.top
```

按照提示完成SSL证书申请（需要输入邮箱）。

完成后，你的API就可以通过 `https://api.成都opc.top` 访问了！

## ❓ 常见问题

**Q: 网站打不开？**
A: 
1. 检查DNS是否生效（可能需要10-30分钟）
2. 检查服务器防火墙是否开放3000端口
3. 检查PM2服务是否运行：`pm2 status`

**Q: API报错？**
A: 
1. 检查后端日志：`pm2 logs chengdu-opc-backend`
2. 检查数据库连接：确认DATABASE_URL正确
3. 检查环境变量：`cat ~/chengdu-opc-/backend/.env`

**Q: 如何备份数据？**
A: 
1. 数据库：Supabase自动每日备份
2. 代码：已在GitHub备份
3. 服务器：阿里云控制台可创建快照

**Q: 如何升级配置？**
A: 在阿里云控制台可以随时升级服务器配置
